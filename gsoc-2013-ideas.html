Ideas

sevntu.checkstyle

Brief explanation:
Extension to EclipseCS plugin, Incubator for Checkstyle checks that are not present in main stream of Checkstyle project. If you have a check and need place to be build-in, let us know.
If you want to study Java(with mentoring) and help us develop some Checks let us know, we are looking for new maintainers and level of Java experience does not matter.
Issue Tracker entries: 21,12,21,21,21,21

Expected results: all issues have to be closed, all code covered with UTs, JavaDoc is created for each class, each Check tested agains huge opensource projects(Spring, Hibernate, Log4j, ...), patches should be created for Checkstyle project.

Knowledge Prerequisite: basic Java (Java SE), basic knowledge of parsers and syntactic analysis.

Eclipse-DSM-viewer

Brief explanation:
Dependency Structure Matrices (DSMs) are very poverful in the analysis of software architecture and design as it allow the developer/architect to easily to see inter-module dependencies than traditional "box and arrows" techniques which become impossible to use for applications that are bigger then 20 classes. DSM already exist in popular IDEs (Idea, MS Visual Studio), and as separate application Latix, NDepend, Sonar. Our idea to create native to Eclipse plugin with free license to let all Eclipse developers to encorporate DSM analysis during development but not as "after commit" analysis. In that milestone we will focus on UI presentation, DSM structure will be taken from opensource library dtangler, additionally we wil focus on convenience to use in Eclipse and navigation from matrix to real code. Detailed description of project could be found here.
Issue Tracker entries: 21,12,21,21,21,21

Expected results: all issues have to be closed, JavaDoc is created for each class, UI rendering support inner tables features as it done in most others applications, Plugin have to be tested agains huge opensource projects(Spring, Hibernate, Log4j, ...).

Knowledge Prerequisite: basic Java (Java SE), SWT, Eclipse RCP.
